HTML vs XHTML

O que é XHTML?

XHTML é uma versão do HTML mais rígida e baseada em XML.

  • XHTML significa Extensible Hyper Text Markup Language
  • XHTML é uma versão de HTML mais rígida e baseada em XML
  • XHTML é HTML definido como um aplicativo XML
  • XHTML é suportado por todos os principais navegadores

Por que XHTML?

XML é uma linguagem de marcação onde todos os documentos devem ser marcados corretamente (ser "bem formados").

O XHTML foi desenvolvido para tornar o HTML mais extensível e flexível para trabalhar com outros formatos de dados (como XML). Além disso, os navegadores ignoram erros nas páginas HTML e tentam exibir o site mesmo que apresente alguns erros de marcação. Portanto, o XHTML vem com um tratamento de erros muito mais rígido.

As diferenças mais importantes do HTML

Diferença Descrição
<!DOCTYPE> é obrigatório O documento XHTML deve ter uma declaração XHTML <!DOCTYPE>
O atributo xmlns em <html> é obrigatório O atributo xmlns em <html> deve especificar o namespace xml do documento
<html>, <head>, <title> e <body> são obrigatórios Esses elementos devem estar presentes em um documento XHTML
Os elementos devem sempre estar adequadamente aninhados Os elementos devem estar corretamente aninhados uns nos outros
Os elementos devem estar sempre fechados Os elementos devem sempre ser fechados corretamente
Os elementos devem estar sempre em letras minúsculas Os nomes dos elementos devem estar em letras minúsculas
Os nomes dos atributos devem estar sempre em letras minúsculas Os nomes dos atributos devem ser escritos em letras minúsculas
Os valores dos atributos devem sempre ser citados Os valores dos atributos devem sempre ser citados
A minimização de atributos é proibida A minimização de atributos não é permitida em XHTML

XHTML - <!DOCTYPE ....> é obrigatório

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd"> <html xmlns="http://www.w3.org/1999/xhtml"> <head> <title>Title of document</title> </head> <body> some content here... </body> </html>

Elementos XHTML devem ser aninhados corretamente

<b><i>Some text</i></b>

Em XHTML, os elementos devem sempre estar adequadamente aninhados uns nos outros, assim:

<b><i>Some text</i></b>

Elementos XHTML devem sempre ser fechados

<p>This is a paragraph</p> <p>This is another paragraph</p>

Elementos vazios XHTML devem sempre ser fechados

A break: <br /> A horizontal rule: <hr /> An image: <img src="happy.gif" alt="Happy face" />

Os elementos XHTML devem estar em letras minúsculas

<body> <p>This is a paragraph</p>

Os nomes dos atributos XHTML devem estar em letras minúsculas
