Sempre declarar o tipo de documento

O tipo de documento correto para HTML é:

<!DOCTYPE html>

Use nomes de elementos em letras minúsculas

HTML permite misturar letras maiúsculas e minúsculas nos nomes dos elementos. No entanto, recomendamos usar nomes de elementos em letras minúsculas porque:

  • Misturar nomes maiúsculos e minúsculos parece ruim
  • Os desenvolvedores normalmente usam nomes em minúsculas
  • Letras minúsculas parecem mais limpas
  • Letras minúsculas são mais fáceis de escrever
<body>
<p>This is a paragraph.</p>

Confira:

This is a paragraph.

Fechar todos os elementos HTML

Em HTML, você não precisa fechar todos os elementos (por exemplo, o <p> elemento). No entanto, recomendamos fortemente fechar todos os elementos HTML.

<section>
<p>This is a paragraph.</p>
<p>This is a paragraph.</p>
</section>

Use nomes de atributos em letras minúsculas

HTML permite misturar letras maiúsculas e minúsculas em nomes de atributos. No entanto, recomendamos usar nomes de atributos em letras minúsculas.

<a href="https://github.com/lautoledo">Visite no github</a>

Confira:

Visite no github

Sempre citar valores de atributos

HTML permite valores de atributos sem aspas. No entanto, recomendamos citar valores de atributos.

<table class="striped">

Sempre especifique alt, largura e altura para imagens

É importante sempre especificar o alt atributo para imagens e definir o width e height.

<img src="html5.gif" alt="HTML5" style="width:128px;height:128px">

Espaços e sinais de igual

HTML permite espaços em torno de sinais de igual. Mas sem espaço é mais fácil de ler e agrupa melhor as entidades.

<link rel="stylesheet" href="styles.css">

Evite longas linhas de código

Tente evitar linhas de código muito longas.

<body>
<h1>Famous Cities</h1>
<h2>Tokyo</h2>
<p>Tokyo is the capital of Japan, the center of the Greater Tokyo Area, and the most populous metropolitan area in the world.</p>
</body>

Linhas em branco e recuo

Não adicione linhas em branco, espaços ou recuos sem motivo.

Para facilitar a leitura, adicione dois espaços de recuo. Não use a tecla Tab.

<body>
<h1>Famous Cities</h1>
<h2>Tokyo</h2>
<p>Tokyo is the capital of Japan, the center of the Greater Tokyo Area, and the most populous metropolitan area in the world.</p>
</body>

Bom exemplo de tabela

<table>
<tr>
<th>Name</th>
<th>Description</th>
</tr>
<tr>
<td>A</td>
<td>Description of A</td>
</tr>
</table>

Confira:

Name Description
A Description of A
B Description of B

Bom exemplo de lista

<ul>
<li>London</li>
<li>Paris</li>
<li>Tokyo</li>
</ul>

Confira:

  • London
  • Paris
  • Tokyo

Nunca pule o elemento <title>

O <title> elemento é obrigatório em HTML.

O conteúdo do título de uma página é muito importante para a otimização de mecanismos de pesquisa (SEO)! O título da página é usado pelos algoritmos do mecanismo de pesquisa para decidir a ordem ao listar as páginas nos resultados da pesquisa.

<title>HTML Style Guide and Coding Conventions</title>

Omitindo <html> e <body>?

Uma página HTML será validada sem as tags <html> e <body>:

<!DOCTYPE html>
<head>
<title>Page Title</title>
</head>
<h1>This is a heading</h1>
<p>This is a paragraph.</p>

Omitindo <head>?

A tag HTML <head> também pode ser omitida. Os navegadores adicionarão todos os elementos antes de <body> a um <head> elemento padrão.

<!DOCTYPE html>
<html>
<title>Page Title</title>
<body>
<h1>This is a heading</h1>
<p>This is a paragraph.</p>
</body>
</html>

Fechar elementos HTML vazios?

Em HTML, é opcional fechar elementos vazios.

<meta charset="utf-8">

Adicione o atributo lang

Você deve sempre incluir o lang atributo dentro da <html> tag, para declarar o idioma da página web.

<html lang="pt-BR">
<head>
<title>Page Title</title>
</head>
<body>
<h1>This is a heading</h1>
<p>This is a paragraph.</p>
</body>
</html>

Metadados

Para garantir a interpretação adequada e a indexação correta do mecanismo de pesquisa, tanto o idioma quanto a codificação de caracteres devem ser definidos o mais cedo possível em um documento HTML:

<!DOCTYPE html>
<html lang="en-us">
<head>
<meta charset="UTF-8">
<title>Page Title</title>
</head>
<body>
<h1>This is a heading</h1>
<p>This is a paragraph.</p>
</body>
</html>

Configurando a janela de visualização

Você deve incluir o seguinte <meta> elemento em todas as suas páginas da web:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Comentários HTML

Comentários curtos devem ser escritos em uma linha:

<!-- This is a comment -->

Comentários que ocupam mais de uma linha devem ser escritos assim:

<!--
This is a long comment example. This is a long comment example.
This is a long comment example. This is a long comment example.
-->

Usando folhas de estilo

Use sintaxe simples para vincular folhas de estilo:

<link rel="stylesheet" href="styles.css">

Acessando elementos HTML com JavaScript

Usar código HTML "desarrumado" pode resultar em erros de JavaScript.

getElementById("Demo").innerHTML = "Hello";
getElementById("demo").innerHTML = "Hello";

Use nomes de arquivos em minúsculas

Alguns servidores web diferenciam maiúsculas de minúsculas em relação aos nomes de arquivos.

london.jpg

Extensões de arquivo

Os arquivos HTML devem ter uma extensão .html.

index.html
